## Summary

The fix ensures that `piWalletQrData` state is **always synchronized** with the user's entered wallet address through a dedicated `useEffect` hook that:

1. **Listens** to changes in `profile.piWalletAddress`
2. **Syncs** the value to `piWalletQrData` immediately
3. **Enables** the QR code dialog to display the correct QR code
4. **Persists** data through auto-save to Supabase

This is a **critical fix** that enables the entire wallet QR code feature to function properly.
